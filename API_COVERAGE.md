# Node Plugin API Coverage

This plugin is a native SA compatibility layer for common Node.js APIs. It does not execute JavaScript and does not depend on Node, V8, or VM APIs.

## Implemented Native Surfaces

- `os`, `process`, `path`, `querystring`, `url`, `punycode`, `util`, `buffer`, `string_decoder` expose native ABI functions and SA macros. `os.constants` now exposes structured signal, errno, priority, dlopen, and libuv constants; `os.getPriority`/`setPriority` use POSIX process priority APIs; `process.kill` sends real POSIX signals; `process.resourceUsage` exposes `getrusage(2)` JSON; `process.availableMemory`/`constrainedMemory` read Linux memory and cgroup limits; `process.features` exposes the native build capability subset; and `util.callbackify/promisify`, `util.parseArgs`, `util.mimeType`, `buffer.resolveObjectURL`, and `fs.opendir`/`opendir_next` have concrete native behavior.
- `fs` exposes sync-style file, directory, fd, metadata, link, glob, and statfs operations. Ownership, fd sync, timestamp, and filesystem-stat calls use POSIX/libc `chown`, `fchown`, `fsync`, `fdatasync`, `futimens`, `utimensat`, and `statvfs` behavior. `fs.promises` exposes promise-module ABI entry points for the same common file, directory, metadata, fd, and `FileHandle`-style operations, returning already-resolved native results to SA code.
- `crypto` exposes random bytes/UUID/int/fill, hash/HMAC state, PBKDF2, HKDF, scrypt, simple cipher/decipher handles, Ed25519 sign/verify, key generation, hash listing, and a Web Crypto sync subset for `getRandomValues`, `randomUUID`, `subtle.digest`-style hashes, raw key import/generation/export, HMAC-SHA256 sign/verify, Ed25519 sign/verify, and AES-256-GCM encrypt/decrypt.
- `net` exposes TCP connect/listen/accept/read/write/end, Unix domain socket path connect/listen with server-close unlink cleanup, server `address()`/`listening` state, close semantics, connection counts, and `maxConnections` metadata/drop behavior including Node v21+ `maxConnections = 0` drop-all semantics, shared `ref`/`unref`/`hasRef` handle state for sockets and servers, local/remote address metadata and Node-style local/remote address/family/port getters, `bytesRead`/`bytesWritten` counters, `bufferSize`/`connecting`/`pending`/`readyState` state helpers, `autoSelectFamilyAttemptedAddresses`-style attempted remote address JSON for synchronous outbound TCP connects, socket `readable`/`writable`/`closed`/`destroyed` state helpers, non-freeing `destroy`, `setTimeout`/timeout metadata backed by socket receive/send timeouts, TCP `resetAndDestroy` via linger-reset close, `setNoDelay`, `setKeepAlive`, send/receive buffer size configuration, `setTypeOfService`/`getTypeOfService` for connected sockets via `IP_TOS`/`IPV6_TCLASS`, write-side shutdown, `SocketAddress` create/parse/address/family/port/flowlabel/json handles, `BlockList` address/range/subnet/check/rules handles including `SocketAddress` handle inputs, outbound TCP connect `BlockList` checks, server inbound `BlockList` filtering with copied rules, common `createConnection(options)` fields via native `family` selection, `localAddress`/`localPort` bind, `noDelay`, `keepAlive`, timeout, and `blockList`, `isIP` helpers, and default auto-select-family/attempt-timeout module settings. `createConnection` now delegates to TCP connect; `createServer` creates a real listener handle on an ephemeral port. `bufferSize` and `connecting` reflect this ABI's synchronous connect/write model and report zero for established sockets; attempted-address metadata records real addresses tried by this synchronous ABI, not a fake Happy Eyeballs race.
- `dgram` exposes UDP4/UDP6 create/bind/send/recv/close, create-with-options for `reuseAddr`/`reusePort`/`ipv6Only`/send and receive buffer sizing, send/receive `net.BlockList` filtering with copied rules, connect/disconnect, connected send, local/remote address metadata, `ref`/`unref`/`hasRef` handle state, broadcast, TTL, send/receive buffer size configuration, send queue size/count queries, IPv4 multicast TTL/loopback/interface/membership/source-specific membership controls, and UDP6 multicast hops/loopback/interface/membership controls. Send queue size/count report `0` because this plugin's UDP sends complete synchronously instead of queuing libuv send requests.
- `dns` exposes lookup/resolve plus resolver-backed record entry points for A, AAAA, ANY, CAA, NAPTR, SOA, TLSA, CNAME, MX, NS, TXT, SRV, PTR, reverse lookup, IPv6 reverse lookup, lookupService, Node-compatible lookup flags/error-code constants, global resolver server configuration, default result order configuration, `lookup()` option handling for `family`, `hints`, `all`, and `order`, and Resolver instance handles with independent server lists/snapshots/common resolve methods. `ADDRCONFIG`, `V4MAPPED`, and `ALL` lookup hints are implemented within the plugin's synchronous OS resolver model. Resolver instances with custom `setServers` now issue UDP DNS queries to the configured server before parsing records, and `setLocalAddress` validates/stores IPv4/IPv6 bind addresses used by those custom UDP queries. `cancel` is exposed as synchronous Resolver state in this ABI and records calls in snapshots; there is no in-flight async DNS operation to interrupt. `dns.promises` exposes the same native results, constants, global server/default-order configuration helpers, and Promise Resolver handles, returning already-resolved buffers to SA code.
- `http` exposes HTTP/1 client, async client, one-shot `request`/`get` JSON helpers, server, streaming response, and WebSocket bridge by compiling the official `http-client` and `http-server` plugin implementations into `libnode.so`; it also exposes Node-compatible `METHODS`, `STATUS_CODES`, `maxHeaderSize` default metadata, `setMaxIdleHTTPParsers()` parser-pool limit state, and low-level header name/value validators. The one-shot helpers return completed native response JSON and do not emulate Node's asynchronous `ClientRequest` event object.
- `http2` exposes native constants JSON, `sensitiveHeaders` metadata, default settings JSON, `getPackedSettings`/`getUnpackedSettings` helpers for RFC wire-format settings payloads, and a cleartext prior-knowledge client request helper through dynamically loaded `libnghttp2`, returning status, headers, body, and frame count as JSON.
- `quic`/`http3` expose native constants, ALPN/RFC metadata, and runtime capability detection for the required `ngtcp2`/`nghttp3`/OpenSSL stack without depending on V8 or Node internals.
- `https` client requests are available through the HTTP client bridge when Zig TLS support is available, including one-shot `request`/`get` JSON helpers for `https://` URLs; `tls` exposes default constants, `DEFAULT_MIN_VERSION`/`DEFAULT_MAX_VERSION`, `convertALPNProtocols()`-style ALPN wire-format encoding, `getCiphers()`-style native cipher listings, native-system `rootCertificates`/`getCACertificates()` JSON, `setDefaultCACertificates()` PEM parsing, `createSecureContext()` handles for CA/cert/key/cipher/version metadata, secure-context snapshots/free, secure-context-backed client verification for custom CA bundles, and a synchronous client socket subset for connect/write/read/close, common `tls.connect(options)` fields via native `family` selection, `localAddress`/`localPort` bind, `noDelay`, `keepAlive`, timeout, and `blockList`, authorization metadata, SNI/servername metadata, ALPN absence metadata, negotiated protocol and cipher metadata, local/remote address/family/port metadata, timeout metadata backed by socket receive/send timeouts, byte counters, `bufferSize`/`connecting`/`pending`/`readyState` inherited socket-state helpers, readable/writable/closed/destroyed state helpers, `ref`/`unref`/`hasRef` handle state, and non-freeing destroy semantics. `bufferSize`, `connecting`, and `pending` reflect this ABI's synchronous TCP connect and TLS handshake model and report zero for established sockets.
- `events`, `stream`, `timers`, `readline`, `console`, `tty`, `perf_hooks`, `diagnostics_channel`, `trace_events`, `async_hooks`, `worker_threads`, `child_process`, `cluster`, `domain`, `deprecated`, `report`, and `sea` expose compatibility handles or status APIs. `events` tracks listener state across base and extra APIs; `stream` and `readline` now use allocated native handles rather than sentinel pointers. `timers.promises` exposes synchronous awaitable-style helpers for setTimeout, setImmediate, scheduler.wait/yield, and setInterval tick handles. `readline.promises` exposes a native Interface handle with buffered input, question, close, snapshot, and free operations. `tty` uses POSIX `isatty`, termios raw-mode toggling, window-size ioctl/env fallback, and environment-aware color-depth helpers. `perf_hooks` exposes monotonic now/timeOrigin, marks/measures entries, histogram sample statistics with percentiles, and process CPU/wall-clock event-loop utilization estimates. `worker_threads` exposes main-thread metadata, environment data, MessageChannel/MessagePort queues, receiveMessageOnPort, and a main-thread postMessageToThread compatibility path. `child_process` executes real subprocesses for exec/execFile/spawn/spawnSync/fork-style helpers, including SA argv-vector parsing and real child pid reporting. `cluster` now exposes primary/worker metadata, scheduling-policy state, `setupPrimary()` metadata, JSON-based `setupPrimary()` config for `exec`/`cwd`/`env`, primary snapshot JSON, subprocess-backed worker fork handles, stdin/stdout message exchange, worker pid/alive/connected snapshots, signal-name kills, wait JSON, exited-after-disconnect state, and disconnect/kill/free helpers. `domain` now exposes native domain handles with explicit `enter()`/`exit()` stack state, member add/remove/count helpers, active-domain lookup, snapshot JSON, dispose, and free helpers without claiming JavaScript callback wrapping semantics. `deprecated` now exposes a native deprecation registry with idempotent code tracking, snapshot/clear/has helpers, and `NODE_OPTIONS`/environment-based deprecation flag introspection without claiming JavaScript wrapper or warning-event semantics.
- `zlib` exposes synchronous gzip/gunzip, deflate/inflate, raw deflate/inflate, unzip, Brotli compress/decompress, Zstd compress/decompress, and CRC32 helpers using `std.compress` plus dynamically loaded Brotli/Zstd system libraries.
- `web_streams` exposes synchronous byte-buffer helpers for readable, writable, and transform stream handles, including enqueue/read/write/snapshot/close/free operations.
- `sqlite` exposes a native synchronous subset through dynamically loaded `libsqlite3`: version, open/close, exec, query JSON, prepare, typed bind int/double/text/blob/null, step, reset, clearBindings, finalize, changes, lastInsertRowid, online backup init/step/remaining/finish plus backup-to-file helpers, session changeset/patchset/isEmpty/memoryUsed/close/applyChangeset helpers, and an SQLTagStore-style prepared statement cache with all/get/run/snapshot/clear/free.
- `ffi` exposes a native dynamic-library helper subset through `std.DynLib`: open/close, symbol existence, common C ABI integer calls, C string length calls, string/integer calls, and string pointer return helpers.
- `vfs` exposes an in-memory `MemoryProvider`-style virtual filesystem subset with mount-like handles, cwd/chdir, mkdir, write/append/read file, file-handle open/read/write/fstat/ftruncate/close, dir-handle opendir/read/close/snapshot, watch/watcher next/snapshot/close event queues, readdir, stat/lstat JSON, exists, unlink, recursive rm, rename, copyFile, symlink/readlink, realpath, provider snapshot, and free helpers.

## Explicitly Excluded

- `vm`, `v8`, `module` loader execution, `repl`, `inspector`, `debugger`, and Node internal bootstrap/realm behavior are out of scope because they require a JavaScript runtime or V8 semantics.

## Remaining Strict Node Gaps

- Full `http2` session/server/TLS ALPN/stream/push runtime semantics, QUIC/HTTP3 endpoint/session/stream/request transport, `tls` socket/server object model, and `dtls` still require mature native protocol library integration.
- Full c-ares channel behavior such as search domains/rotation/cancelation, true JavaScript Promise object identity for `dns.promises`, and full Node DNS error-code semantics still need c-ares-level compatibility.
- `events`, `streams`, `timers`, `readline`, `worker_threads`, `cluster`, `domain`, and `test` need a shared event loop and native callback dispatch model for strict Node behavior; strict timer ref/unref, AbortSignal cancellation, Promise object identity, TTY raw-mode behavior, WHATWG Web Streams locking, backpressure, reader/writer, pipe, tee, and async iteration semantics are not complete. `cluster` currently models real subprocess workers and simple stdin/stdout messaging only; it does not yet expose Node's EventEmitter worker/server objects, shared-listener distribution, or internal IPC serialization. `domain` currently models explicit native domain stack handles and membership metadata only; it does not yet expose EventEmitter inheritance, bind/intercept wrappers, or async callback propagation across other module callbacks.
- `wasi`, real OS-backed VFS mounts and complete VFS stream/watch async/event-emitter APIs, full libffi type/signature varargs/callback support, SQLite session filter/conflict callback semantics and strict JS SQLTagStore template-literal semantics, and full Web Crypto RSA/ECDSA/ECDH/KDF/wrapKey/unwrapKey/JWK/structured `CryptoKey` semantics need first-class ABI surfaces and tests.
- Node-compatible error classes/codes and option validation are not yet complete across modules.
